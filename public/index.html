The NC News Database - A RESTful API for a news articles database.
What's it all about? Allow me to explain...

What's inside?!
The data itself can be broken down into the following categories:

  • Topics
  • Articles
  • Users
  • Comments

What can it do?!
Methods of Database Interaction

Currently, information within the database can be accessed, added to, modified, and deleted.
This is done using HTTP methods, which allow you to send and retrieve data to and from the server.
Let's take a look at these different methods...

  1. GET /api/topics
    Description:
    Retrieves a list of all topics available.
    Response:
    An array of topic objects.
    Example: {
    "slug": "football",
    "description": "Footie!"
    }

  2. GET /api/articles
    Description:
    Fetches a list of all articles available.
    Response:
    Array of article objects.
    Example: {
    "title": "Seafood substitutions are increasing",
    "topic": "cooking",
    "author": "weegembump",
    "created_at": "2018-05-30T15:59:13.341Z",
    "votes": 0,
    "comment_count": 6
    }

  3. GET /api/users
    Description:
    Retrieves all users in the database.
    Response:
    Array of user objects.
    Example: {
    "username": "butter_bridge",
    "name": "jonny",
    "avatar_url": ""
    }

  4. GET /api/articles/:article_id
    Description:
    Returns a specific article by its article_id.
    Response:
    An article object.
    Example: {
    "title": "Living in the shadow of a great man",
    "topic": "mitch",
    "author": "butter_bridge",
    "body": "I find this existence challenging",
    "created_at": "2020-07-09T20:11:00.000Z",
    "votes": 100,
    "article_img_url": ""
    }

  5. GET /api/articles/:article_id/comments
    Description:
    Fetches all comments for the specified article ID.
    Response:
    Array of comment objects.
    Example:{
    "comment_id": 89,
    "article_id": 1,
    "body": "Esse et expedita harum non. Voluptatibus commodi voluptatem.
    Minima velit suscipit numquam ea.
    Id vitae debitis aut incidunt odio quo quam possimus ipsum.",
    "votes": 2,
    "author": "cooljmessy",
    "created_at": "2020-10-24T06:08:00.000Z"
    }

  6. POST /api/articles/:article_id/comments
    Description:
    Adds a new comment to the specified article.
    Request body:{
    "username": "example_user",
    "body": "This is a comment."
    }
    Response:
    The newly created comment object.
    Example: {
    "username": "example_user",
    "body": "This is a comment."
    }

  7. PATCH /api/articles/:article_id
    Description:
    Updates the vote count for a specific article (incrementing or decrementing).
    Request body: {
    "inc_votes": 1
    }
    Response:
    The updated article object with new vote count.
    Example: {
    "title": "Living in the shadow of a great man",
    "topic": "mitch",
    "author": "butter_bridge",
    "body": "I find this existence challenging",
    "created_at": "2020-07-09T20:11:00.000Z",
    "votes": 105,
    "article_img_url": ""
    }

What could go wrong?!
Errors and what they mean.

Status Reason
400 Bad Request (e.g., invalid ID format)
404 Path Not Found (e.g., ID does not exist)
500 Internal Server Error (you cannot helped, good luck)
